现在很多编程语言都支持类型别名，比如 type Identity = int32，然后就可以在代码中使用 Identity 来表示 int32 了。很多同学都觉得这种写法有点脱裤子放屁，没啥必要。还有一些同学认识到了这种方式能够达到代码更好的可读性的效果，你可以直接从代码中理解这个 int32 是干嘛用的。其实这种别名还有一个更重要的功能：类型隐藏。假设你现在没有使用这种别名，然后你可能在各个需要 id 的地方返回了 int32，突然由于某种原因，你需要让 id 变成 string 类型。这时候你的 int32 类型已经侵入了你的实现代码的各个部分。而使用了类型别名之后，你的修改就非常可控了，这种信息隐藏帮你隐藏了变化源。其实这种类型别名并不是什么新鲜事，想想 c 里面的宏，就在预编译阶段解决了这个问题。
