nginx 在 linux 上使用了 sendfile 系统调用，这个系统调用可以将磁盘上的数据直接发送到网络上（不需要通过用户态）

nginx 与 apache 相比最大的优势在于减少了上下文切换和内存占用

上下文切换：内核切换 CPU 让其在不同的地址空间上做操作。中断：由物理设备发送给内核的信号，通常是请求 I/O 服务。

在高并发竞态场景下使用 CAS (compare-and-swap) 可以防止被 block 后挂起再恢复，这能有效减少不必要的上下文切换，提高 performance。其实和数据库乐观锁是一个道理。在很多类似场景下，使用 atomic 要优于 synchronized

为什么像 goroutine 或者其他一些协程实现在性能上要优于多线程（这里指系统线程而非用户线程）？本质原因是协程是用户态的，或者说协程是在用户态进行调度的，因此其上下文切换非常之快。但是也正因如此，协程只能跑在一个核上，而且一旦某个协程被阻塞，可能导致整个进程不可用。（但是 goroutine 却是可以用多核的，因为 golang 在运行时会起多个线程，并把 goroutine 调度到线程上）

内核的进程调度器维护着各种状态的进程队列，包括一个所有可运行进程的队列，以及一个包括所有休眠进程和僵尸进程的列表。进程调度器的一个重要工作就是决定下一个运行的进程。
